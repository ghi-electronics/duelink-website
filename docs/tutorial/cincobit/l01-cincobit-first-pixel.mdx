# L1: First Pixel

notes: click blocks to execute.

## Overview

In this lesson, students will connect CincoBit or [PixoBit](https://www.duelink.com?utm_source=chatgpt.com) to a computer and begin programming using [MicroBlocks](https://microblocks.fun?utm_source=chatgpt.com).

Students will learn how MicroBlocks communicates with the board in real time and how code changes can instantly update the device without needing to recompile or upload a full program.

Using the display, students will light pixels, create simple images, and explore how computers use small points of light to create graphics and animations.

---

# Learning Goals

Students will:

* connect a board using USB
* connect MicroBlocks to the device
* understand live programming updates
* light LEDs and create simple graphics
* explore coordinates and pixel displays

# What is MicroBlocks?

MicroBlocks is a live block-based programming environment designed for physical computing devices.

Unlike traditional systems where programs must be compiled and uploaded every time a change is made, MicroBlocks updates the board instantly while the program is running.

This allows students to:

* experiment quickly
* test ideas immediately
* learn through exploration

Changes can happen live while the device is operating.

---

# Step 1 — Connect the Device

1. Plug CincoBit or PixoBit into the computer using USB.
2. Open MicroBlocks.
3. Click the connect button.
4. Select the board from the list.

Once connected, MicroBlocks can directly communicate with the device.

---

# Step 2 — Test Live Updates

Create a simple program that turns on part of the display.

Change the program while it is running and observe how the device updates immediately.

Try:

* changing a pixel
* changing a number
* changing a delay
* changing text

Notice:

> The board updates instantly without restarting the program.

This is called **live programming**.

# Understanding Pixels

Displays are made from many tiny lights called **pixels**.

* CincoBit uses a 5×5 LED matrix.
* PixoBit uses a larger graphical display with many more pixels.

Programs control pixels by turning them on or off to create:

* images
* text
* animations
* games

# Troubleshooting

## Device does not appear

* reconnect USB cable
* try another cable
* restart MicroBlocks

## Program does not respond

* verify the board is connected
* reconnect the device

## Display looks incorrect

* check pixel positions carefully
* clear the screen before drawing again
