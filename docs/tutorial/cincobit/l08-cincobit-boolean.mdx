# L8: Boolean

## Overview

In this lesson, students will learn how computers make decisions using Boolean logic.

Boolean logic uses only two values:

* TRUE
* FALSE

These values are used everywhere in computing to represent:

* ON and OFF
* YES and NO
* PRESSED and NOT PRESSED

Students will learn:

* Boolean variables
* TRUE and FALSE conditions
* comparison operators
* `AND` and `OR` logic
* combining multiple conditions together

Using buttons, touch sensing, sounds, and displays, students will create interactive programs that react differently depending on conditions.

---

# Learning Goals

Students will:

* understand Boolean values
* use TRUE and FALSE conditions
* use Boolean variables
* use comparison operators
* combine conditions using `AND`
* combine conditions using `OR`
* display Boolean results using `say`

# What is a Boolean?

A Boolean value has only two possible states:

| Boolean | Meaning             |
| ------- | ------------------- |
| TRUE    | Yes / On / Active   |
| FALSE   | No / Off / Inactive |

Examples:

* light ON or OFF
* button pressed or not pressed
* correct or incorrect answer

Computers constantly use Boolean logic internally.

---

# Understanding TRUE and FALSE

Conditions always produce:

* TRUE
  or
* FALSE

Examples:

| Question              | Result        |
| --------------------- | ------------- |
| Is 5 greater than 3?  | TRUE          |
| Is 2 greater than 10? | FALSE         |
| Is button A pressed?  | TRUE or FALSE |
| Is Pin 1 touched?     | TRUE or FALSE |

Programs use these answers to decide what to do.

---

# Boolean Variables

Variables can store Boolean values.

Example:

```text id="q7m2vk"
alarmOn = true
```

or:

```text id="h4x8rp"
alarmOn = false
```

Programs can change Boolean variables while running.

---

# Displaying Boolean Values

Use the `say` block to display:

* TRUE
* FALSE

Example:

```text id="z8v5mt"
say alarmOn
```

Students should observe how the displayed value changes when the variable changes.

---

# Comparison Operators

Programs compare values using comparison operators.

| Operator | Meaning      |
| -------- | ------------ |
| `=`      | equal to     |
| `>`      | greater than |
| `<`      | less than    |

Examples:

| Expression         | Result                          |
| ------------------ | ------------------------------- |
| `score > 10`       | TRUE if score is larger than 10 |
| `temperature < 20` | TRUE if temperature is below 20 |
| `count = 5`        | TRUE only when count equals 5   |

---

# Understanding AND

The `AND` operator requires:

> BOTH conditions must be TRUE

Examples:

| Condition A | Condition B | Result |
| ----------- | ----------- | ------ |
| TRUE        | TRUE        | TRUE   |
| TRUE        | FALSE       | FALSE  |
| FALSE       | TRUE        | FALSE  |
| FALSE       | FALSE       | FALSE  |

Example:

```text id="m5q2wr"
if buttonA pressed AND buttonB pressed
```

Both buttons must be pressed together.

---

# Understanding OR

The `OR` operator requires:

> ONLY ONE condition must be TRUE

Examples:

| Condition A | Condition B | Result |
| ----------- | ----------- | ------ |
| TRUE        | TRUE        | TRUE   |
| TRUE        | FALSE       | TRUE   |
| FALSE       | TRUE        | TRUE   |
| FALSE       | FALSE       | FALSE  |

Example:

```text id="k8t4py"
if buttonA pressed OR buttonB pressed
```

Either button activates the condition.
