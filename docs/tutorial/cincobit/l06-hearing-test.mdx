# L6: Hearing Test


## Overview

In this lesson, students will combine everything they have learned so far to build an interactive hearing test device.

The program will:

* play tones through the buzzer
* increase the sound frequency step-by-step
* allow the user to stop the test using a button press
* display the final frequency on the screen

Students will use:

* variables
* loops
* conditions
* button events
* timing
* sound generation

to create a complete interactive project.

# What is Frequency?

Frequency describes how fast a sound wave vibrates.

Frequency is measured in:

* Hertz (Hz)

Examples:

| Frequency | Sound        |
| --------- | ------------ |
| 100 Hz    | Low pitch    |
| 1000 Hz   | Medium pitch |
| 5000 Hz   | High pitch   |

Higher frequencies sound higher in pitch.

---

# Human Hearing

Humans can hear many different frequencies, but hearing ability changes with:

* age
* environment
* volume
* hearing health

This project explores:

> How high of a sound can you hear?

# Project Goal

The hearing test will:

1. start at a lower frequency
2. slowly increase the pitch
3. continue playing sounds in a loop
4. stop when the user presses a button
5. display the highest heard frequency

---

# Step 1 — Create Variables

Create a variable called:

```text id="z7k3pm"
frequency
```

Set the starting value:

```text id="m4q8tx"
frequency = 500
```

Create another variable:

```text id="w2v9cs"
stopTest
```

Set:

```text id="0k6nfr"
stopTest = false
```

This variable will help control when the loop stops.

---

# Step 2 — Start the Test

Use:

```text id="3q1xpu"
when button pressed
```

to start the hearing test.

Inside the event:

* reset the variables
* begin the frequency sweep

---

# Step 3 — Create the Sweep Loop

Use:

```text id="h8c5rd"
repeat until stopTest = true
```

Inside the loop:

1. play the current frequency
2. wait briefly
3. increase the frequency

Example:

```text id="f2m9vk"
frequency = frequency + 200
```

Each loop increases the pitch.

---

# Understanding Frequency Changes

As the variable increases:

| Frequency | Result              |
| --------- | ------------------- |
| 500 Hz    | Low tone            |
| 1000 Hz   | Higher tone         |
| 3000 Hz   | Very high tone      |
| 8000 Hz   | Extremely high tone |

Students will hear the pitch rise over time.

---

# Step 4 — Stop the Test

Create another button event.

When the button is pressed again:

```text id="k5r7bn"
stopTest = true
```

Now the loop condition becomes TRUE and the loop stops.

This combines:

* variables
* conditions
* events
* loops

into one complete system.

---

# Step 5 — Display the Result

After the loop ends:

* display the final frequency
* show the number on the screen

This tells the user:

> the highest frequency they could still hear

# Understanding Program Flow

The program follows this sequence:

1. Start test
2. Play sound
3. Increase frequency
4. Repeat loop
5. Wait for button press
6. Stop loop
7. Display result

This is an example of a complete interactive algorithm.

# Troubleshooting

## No sound

* verify buzzer blocks are connected properly
* check frequency values

## Frequency changes too quickly

* increase wait time

## Test never stops

* verify the stop variable changes correctly

## Display does not show the result

* ensure display code runs after the loop ends
