## Overview

In this lesson, students will learn how computers make decisions using Boolean logic.

Boolean logic uses only two values:

* TRUE
* FALSE

These values are used everywhere in computing to represent:

* ON and OFF
* YES and NO
* PRESSED and NOT PRESSED

Students will learn:

* Boolean variables
* TRUE and FALSE conditions
* comparison operators
* `AND` and `OR` logic
* combining multiple conditions together

Using buttons, touch sensing, sounds, and displays, students will create interactive programs that react differently depending on conditions.

## What is a Boolean?

A Boolean value has only two possible states:

| Boolean | Meaning             |
| ------- | ------------------- |
| TRUE    | Yes / On / Active   |
| FALSE   | No / Off / Inactive |

Examples:

* light ON or OFF
* button pressed or not pressed
* correct or incorrect answer

Computers constantly use Boolean logic internally.

---

## Understanding Conditions

Conditions, like when we use `if` block always produce:

* TRUE
  or
* FALSE

Examples:

| Question              | Result        |
| --------------------- | ------------- |
| Is 5 greater than 3?  | TRUE          |
| Is 2 greater than 10? | FALSE         |
| Is button A pressed?  | TRUE or FALSE |
| Is Pin 1 touched?     | TRUE or FALSE |

Programs use these answers to decide what to do.

## Comparison Operators

Programs compare values using comparison operators.

| Operator | Meaning      |
| -------- | ------------ |
| `=`      | equal to     |
| `>`      | greater than |
| `<`      | less than    |

Those work with numbers and result with a boolean.

Examples:

| Expression         | Result                          |
| ------------------ | ------------------------------- |
| `score > 10`       | TRUE if score is larger than 10 |
| `temperature < 20` | TRUE if temperature is below 20 |
| `count = 5`        | TRUE only when count equals 5   |

But you can't use `>` with a boolean. Think about it, how can true be `>` true?

Using `=` makes sense

## Boolean Operators

There are operators that are made for boolean values. The first one is `not`. This results in the opposite result of a variable. Like when you say "if a pin is touched" vs "if the pin is not touched"

![bool not](pathname:///img/tutorials/cincobit-lessons/bool-not.png)

The `and` operator requires:

> BOTH conditions must be TRUE

Examples:

| Condition A | Condition B | Result |
| ----------- | ----------- | ------ |
| TRUE        | TRUE        | TRUE   |
| TRUE        | FALSE       | FALSE  |
| FALSE       | TRUE        | FALSE  |
| FALSE       | FALSE       | FALSE  |

This is useful when having multiple conditions for something to happen.

When a button is pressed `and` lives is less than 1 then game over. Both of these conditions must be met for a "game over"

![bool and](pathname:///img/tutorials/cincobit-lessons/bool-and.png)

The `or` operator requires:

> ONLY ONE condition must be TRUE

Examples:

| Condition A | Condition B | Result |
| ----------- | ----------- | ------ |
| TRUE        | TRUE        | TRUE   |
| TRUE        | FALSE       | TRUE   |
| FALSE       | TRUE        | TRUE   |
| FALSE       | FALSE       | FALSE  |


Example:

If button `A` is pressed `or` button `B` is pressed then play a tone.

![bool or](pathname:///img/tutorials/cincobit-lessons/bool-or.png)
