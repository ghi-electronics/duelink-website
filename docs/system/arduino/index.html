Arduino

This is an interesting option because it allows a DUELink to be used in 2 very different ways. You can program any of the modules using the Arduino IDE/software. Or, you can use an Arduino board to control a stream of Daisylinked DUELink modules. We will cover them individually.

Programming Modules with Arduino​

The chip used on DUELink modules is STM32C071, which is fully supported by the Arduino IDE through the ST extension.

While you can do this on any module, we recommend using DueDuino for a first-class experience with Arduino and DUELink.

Now, you need to completely wipe out the DUELink firmware from the STM32C071 chip and then it's all yours to use with anything, including Arduino! Details on how to accomplish this is on the Loader page.

tip

You can do a complete erase on any DUELink module and then reprogram it with the DUELink software any time you like, and as many times as you like! Everything you need is explained on the Loader page.

Start the Arduino IDE, and install the STM32 MCU based boards extension under Board Manager. More details are found here.

Under Tools/Board/STM32 MCU based boards: Select Nucleo-64. MMMMMMMMMMMMMMMMMMMMMMMMMMM

Under Tools/Board part number: Select Nucleo C071RB.

And finally under Tools/Upload method: Select STM32CubeProgrammer(DFU) for file upload.

Add this code to blink the status LED, which is connected to pin PB8.

void setup() {
  // initialize digital pin as an output.
  pinMode(PB8, OUTPUT);
}

void loop() {
  digitalWrite(PB8, HIGH);  // turn the LED on (HIGH is the voltage level)
  delay(1000);                      // wait for a second
  digitalWrite(PB8, LOW);   // turn the LED off by making the voltage LOW
  delay(1000);                      // wait for a second
}

Assuming you have already a wiped out board with no DUELink Loader present, Put the board/chip in DFU mode by connecting BOOT0/LDR pin to 3.3V. On DUELink modules that have LDR or "A" buttons, you only need to press the button while reset (or power cycle)

If no button is found, there is a special 2-hole-pads with BOOT0/LDR and 3.3V. Use a paper clip (or a wire) to short the two pads.

Reset the boards, and now it is in DFU mode. You can verify by checking the Device Manager.

You can now send the program from the Arduino IDE

The module is all yours now, with the fully Arduino experience.

info

If you'd like to go back to using DUELink software you'll need to reload its firmware. Details are found on the Loader page.

Arduino boards with DUELink Streams​

Any board running Arduino software can utilize DUELink modules, it just needs a board with a JST connector. This is the Downstream connection. An example can be Arduino Uno R4 WiFi, which already include a JST connector.

The Sparkfun Redboard Qwiic is another option.

We recommend using our DueDuino for a hybrid first-class experience with Arduino and DUELink.

tip

If using DueDuino, see the previous section on "Programming Modules with Arduino".

You are now ready to access a stream of Daisylinked modules that are connected to the Downstream socket.

Here is the Arduino Uno R4 WiFi & Sparkfun Redboard Qwiic connected to a 2.3" color display with capacitive touch screen.

We are still in the process of implementing Arduino libraries for you to use. However, the commands are simple text-based. Here is an example that uses the standard Arduino libraries to send LED command downstream to blink that module's status LED 10 times.

// Start I2C
Wire.begin();
//...
//...

// Send LED command
Wire.beginTransmission(0xA4);
Wire.write("LED(200,200,10)");
Wire.endTransmission();
