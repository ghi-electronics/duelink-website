# On-Module Code

Every DUELink module ships with firmware that responds to plain text commands from a host — that's the default workflow, covered in [Get Started](../start). But the same on-board STM32 microcontroller can also run your own code, for projects that don't need a host or where you want full control of the hardware. Here are the ways to do it.

<div style={{ textAlign: 'center'}}>
![Ghizzy eyes — a DUELink module running on its own](pathname:///img/ghizzy-eyes.gif)
</div>

**Quick guide:** MicroBlocks for classrooms · Arduino IDE if you've used Arduino · C++ for full toolchain control. The internal scripting engine that powers the default firmware is also documented under [Advanced](../engine/intro) for the power user.

## Behind the scenes: the Internal Engine

<div style={{ textAlign: 'center'}}>
![DUELink Logo](pathname:///img/duelinklogo.webp)
</div>

The default firmware on every module is a small DUELink scripting engine — it's what receives plain text commands and runs them. You don't usually need to think about it, but advanced users can dig into the engine, scripting language, drivers, host mode, the console, and more under the Advanced section of the docs.
