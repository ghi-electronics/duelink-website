# On-Module Code

Every DUELink module has an onboard STM32 microcontroller (MCU) that you can program directly. Your code lives on the module and runs **standalone** — no host computer needed at runtime.

<div style={{ textAlign: 'center'}}>
![Ghizzy Eyes](pathname:///img/ghizzy-eyes.gif)
</div>

There are a few ways to do this, depending on who you are and what you need.

---

## In the classroom? Use MicroBlocks

<div style={{ textAlign: 'center'}}>
![MicroBlocks](pathname:///img/microblocks.webp)
</div>

For educators and first-time coders, MicroBlocks is the easiest path — block coding that runs directly on the module, with a complete CincoBit curriculum.

## Going deeper: the Internal Engine

<div style={{ textAlign: 'center'}}>
![DUELink Logo](pathname:///img/duelinklogo.webp)
</div>

DUELink ships with an internal scripting engine that runs on every module. You can write full programs in our scripting language and load them on the module to run on their own. This is the most advanced path — see the [Internal Engine](../engine/intro) docs to learn more.
