# MicroBlocks

---

![MicroBlocks](/img/microblocks.png)

For users who enjoy block-coding, MicroBlocks supports several boards. All you need is a board with a JST connector to utilize hundreds of DUELink modules. There are 2 ways to benefit from MicroBlocks, by using a supported third-party board such as [micro:bit](../system/microbit) or by loading MicroBlocks directly on a DUELink module!

## Using micro:bit

We will focus on using [micro:bit](../system/microbit) While any board should be able to work DUELink, . In the case of [micro:bit](../system/microbit), you need an accessory that adds a JST connector, like the one we use for [Downstream](../interface/downstream). For example, [Sparkfun Qwiic micro:bit Breakout](https://www.sparkfun.com/sparkfun-qwiic-micro-bit-breakout.html) is a good option.

![micro:bit-breakout](/img/microbit-breakout.png)

This board connects the JST connector to [I2C](../interface/i2c) bus, which is one of supported [Interfaces](../interface/intro). Now we can connect DUELink modules [Downstream](../interface/downstream).

![micro:bit](/img/microbit-sparkfun-duelink.png)   

Lets get started! Create a MicroBlocks project like you always did on [microblocks.fun/run/](https://microblocks.fun/run/).

![MicroBlocks IDE](/img/microblocks-ide.png)

Connect a micro:bit and add blocks that send `LED(200,200,50)` over the I2C Interface. 

![MicroBlocks Code Blocks](/img/microblocks-blocks.png)

This will blink the LED on a connected DUELink module 50 times.

![Blink Status LED](/img/status-led.gif)

:::tip
A DUELink extension will be provided in the future to make the use of MicroBlocks easier.
:::

## Modules running MicroBlocks

MicroBlocks can run on any of the DUELink modules, giving that module a native block-coding option. From there, a user can easily [Daisylink](../engine/daisylink) several modules.

image showing module with MicroBlocks on it connected to other modules.

While MicroBlocks can be loaded on the entire [Catalog](../catalog/intro.mdx), it shines best on one of the [Microcomputers](../catalog/microcomputers), such as [CincoBit](../catalog/microcomputer/cincobit) and [PixoBit](../catalog/microcomputer/pixobit).


![CincoBit & PixoBit](/img/cincobit-pixobit.png)

You need to replace the default DUELink firmware with the MicroBlocks firmware. You only need this done once! Put your board in [Loader](../loader) mode. If using CincoBit or PixoBit, simply hold the `A` button while resetting the board.

![Putting Module in Loader mode](/img/ldr-a-pads.png)

Click firmware update on MicroBlocks.

![MicroBlocks firmware](/img/microblocks-firmware.png)

Congrats, this board is now running MicroBlocks. Enjoy th entire system and enjoy [Daisylink](../engine/daisylink) connecting other modules to the [Downstream](../interface/downstream) connector!

:::tip
The modules in the chain do not need to be updated with the MicroBlocks firmware. They need the default DUELink firmware to keep working and [Daisylinking](../engine/daisylink) properly!
:::
