# Languages

DUELink modules speak plain text commands over USB, I2C, or UART, so any environment that can write text can drive them. The pages below cover how to use DUELink from each language we officially support — same command names everywhere; the only thing that changes is the library wrapper.

</div>

---

## From your host

Send commands from code running on a PC, Raspberry Pi, Arduino, micro:bit, phone, or any other host.

- **[Python](./python)** — the most common starting point on a PC or Raspberry Pi.
- **[JavaScript](./javascript)** — Node.js on a PC, or directly in the browser via Web Serial.
- **[.NET](./dotnet)** — C# / F# on Windows, macOS, or Linux.
- **[MicroPython](./micropython)** — runs on small MCU boards like the Raspberry Pi Pico.
- **[MakeCode](./makecode)** — block environment for micro:bit and Arduino in the classroom.

---

## On the module

These languages let you write code that runs *on* the DUELink module's on-board STM32, with no host needed.

- **[MicroBlocks](./microblocks)** — block coding aimed at classrooms (full guide under [For Educators](../educators)).
- **[Arduino IDE](./arduino-ide)** — Arduino sketches running on the module's MCU (covered under [On-Module Code](./standalone)).
- **[C++](./cpp)** — bare-metal C++ for full toolchain control.

---

## Niche and experimental

Other places DUELink has been driven from — starting points more than full guides.

- **[App Inventor](./app-inventor)** — control modules from an Android app.
- **[Swift](./swift)** — iOS / macOS.
- **[Excel](./excel)** — yes, from spreadsheet formulas.
- **[Other](./other)** — anything else that can write text over a serial port.
