# Languages

DUELink modules are self-contained microcomputers, allowing them to work in two distinct ways. First, module(s) can be remotely controlled by commands coming from a Tethered [Supported Hardware](../hw/intro). 


<div style={{ textAlign: 'center'}}>
![Supported Hardware](pathname:///img/duelink-supported-hardware.webp)
</div>

The Supported Hardware can Tether to DUELink modules using USB, I2C, or UART. We provide libraries for many coding languages, including [Python](./python), [MicroPython](./micropython), [JavaScript](./javascript), and [.NET](./dotnet.mdx). 


<div style={{ textAlign: 'center'}}>
![languages](pathname:///img/languages.webp)
</div>

The other option is by loading code can on modules directly to run [Standalone](./standalone). This for example, can be using [Arduino](../hw/arduino).

<div style={{ textAlign: 'center'}}>
![MicroBlocks](pathname:///img/arduino-logo.webp)
</div>

Similar to Arduino, [MicroBlocks](./microblocks) can also be used to directly program modules to run Standalone, but this time using blocks!

<div style={{ textAlign: 'center'}}>
![MicroBlocks](pathname:///img/microblocks-sample.webp)
</div>




Modules can also be daisylinked together. Learn about how DUELink [Daisylink](../engine/daisylink) work.

<div style={{ textAlign: 'center'}}>
![Daisylink](pathname:///img/daisylink-normal.webp)
</div>
