## New to Coding

If you are new to coding, or to coding circuits, and not sure what option to pick then we recommend MicroBlocks for a user-friendly block-coding option. As for the hardware, any of the products mentioned on the [Educators](./educators) page will do a great job but we recommend the ultra low-cost [CincoBit](./products/mccincobit-e). If you are an educator, consider the [CincoBit Classroom Kit](./products/ktcincobit-e), which includes everything your classroom needs to start coding circuits for under $100... you only need a Chromebook or Computer!!

## Starting with No Code!  

You can try out the module without needing to write any code, using the online demo. [demo.duelink.com](https://demo.duelink.com/) is an excellent [JavaScript](./language/javascript) demo but you can use it to verify modules functionality, without any coding!

Connect your module to USB on your Computer. If your board does not have a USB connector then use an adapter, like USB Hook.

<div style={{ textAlign: 'center'}}>
![DUELink Connection](pathname:///img/duelink-connection.webp)
</div>

Go to the [loader page](https://loader.duelink.com/update) and make sure your board is loaded with the latest firmware and drivers.

Open the [demo site](https://demo.duelink.com/) and click on the USB logo to connect your device.

<div style={{ textAlign: 'center'}}>
![Demo Connect](pathname:///img/demo-connect.webp)
</div>

The first tab is for controlling the STAT LED. This is an LED that is found on all DUELink modules. Change the parameters and observe the STAT LED on the connected board.

<div style={{ textAlign: 'center'}}>
![Demo LED tab](pathname:///img/demo-led-tab.webp)
</div>

If you are using a board with `A` or `LDR` buttons, you can also try Digital read on these buttons. These buttons are connected to pin `20`. You will need to set `Pull` to `Down`. You can now hold the button up or down while clicking `Read`.

<div style={{ textAlign: 'center'}}>
![Read Button](pathname:///img/demo-digital-read.webp)
</div>

By the way, when connected, the demo site shows the device ID (module type) and its firmware version.

<div style={{ textAlign: 'center'}}>
![Device Firmware](pathname:///img/demo-firmware-version.webp)
</div>
