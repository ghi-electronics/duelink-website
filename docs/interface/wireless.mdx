# Wireless


Cut the wire and use DUELink modules wirelessly! This **wireless interface** is done using the [Wireless ESP32](../products/wlesp32-a) module. 

<div style={{ textAlign: 'center'}}>
![Wireless daisylink](pathname:///img/catalog/wlesp32-a-1.webp)
</div>

:::tip
Wireless ESP32 module can also be used as a standard module with Wireless connection, without becoming an interface, see [IoT WiFi](../iot/wifi) page.
:::

Either WiFi or Bluetooth can be used to enable the Wireless Interface to access the Daisylink wirelessly.

<div style={{ textAlign: 'center'}}>
![Wireless daisylink](pathname:///img/wireless-daisylink.webp)
</div>

In either way, the Wireless ESP32 module needs to be loaded with a special `Wireless Bridge` script.

1. Connect the modules to a PC using any USB adaptor, such as [Power Hook](../products/adpwrhook-a) [USB Host](../products/adusbhook-a).

<div style={{ textAlign: 'center'}}>
![Wireless power options](pathname:///img/wlesp32-power-options.webp)
</div>

2. Open [console.duelink.com](https://console.duelink.com/) and click the `Connect` button and then on the right side, click `Erase All`. The erase all step is needed to remove all regions.

<div style={{ textAlign: 'center'}}>
![Wireless power options](pathname:///img/erase-all.webp)
</div>

3. Copy/paste this [Wireless Bridge script](https://github.com/ghi-electronics/duelink-projects/blob/main/source/wireless-bridge/standalone/wireless-bridge.txt) into the console.
4. Modify the configuration as explained below.
5. Click record and cycle power.

Note how Wireless ESP32 is a wireless bridge connecting wireless to the chain and therefore its DUELink address 1 is only visible on the USB connection. All other modules are no longer accessible from USB as they are now accessible from Wireless.

<div style={{ textAlign: 'center'}}>
![Wireless daisylink address](pathname:///img/wireless-daisylink-addr-host.webp)
</div>

The bridge script configuration allows you to select between WiFi and Bluetooth. The default option is Bluetooth, which is recommended. This will turn the module into a virtual serial port connection using the Bluetooth SPP profile. This allows a PC running one of the languages, such as [Python](../language/python), to access the connected modules wirelessly. Everything works exactly the same as if you were using a USB virtual serial connection.

```py
# Connect a Daisylink bridge using Bluetooth
StartBT("DUELink_BT")
# Wait for Bluetooth connection and initiate the bridge
while (dread(5,1)=1)
    wait(100)
wend

# Initiate the bridge
Bridge(1)
```

WiFi works similarly. It connects to a WiFi network and start a TCP server at port 22.

```py
# Connect a Daisylink bridge using WiFi
StartWiFi("MySSID", "MyPassword") 

# Wait for multicast DNS connection
StartTcp("DUELink")
while (dread(4,1)=1)
    wait(100)
wend
# Initiate the bridge
Bridge(1)
```

Note that provided libraries only work with serial ports. The easiest way to try this server is using a terminal software, such as TeraTerm. Open TeraTerm and select `TCP/IP` then enter `DUELink` for the `Host` name. Select `Other` from the `Service` options. Set `TCP Port#` to `22`. Once the connections is established, you can type in commands to control modules. Go ahead and enter `list` and hit enter to see the code listing.

:::tip
`Echo()` does not work properly as TeraTerm (and most terminals) will only send data when you hit enter when used in TCP mode. You can enable `local echo` to see what you type!
:::
