Standard Library

These standard library functions are built in natively into the firmware and are available on all systems. You will likely not use these directly, but use the simpler wrapper methods provided by the individual modules.

tip

These are particularly useful for using Mainboards or or for making custom modules.

Special Pins​

Some functions only work on pins with specific features.

Analog capable pins: P1, P2, P3, P4, P5, P6, P7, P8, P9, P17 Interrupt capable pins: P1, P2, P3, P4, P5, P6, P7, P12 PWM capable pins: P1, P2, P3, P4, P5, P6, P7, P8, P11

tip

PWM pins share timers. Changing the frequency on one will effect the other. Changing duty-cycle is okay!

  • P1, P2
  • P3
  • P4, P11
  • P5, P6, P7, P8

Pin Control​

LED​

Control the on-board STAT LED, found on every single DUELink module.

FunctionDescription
LED(high, low, count)high: duration on in milliseconds, low: duration off in milliseconds, count: number of times to repeat, -1 to run forever.

Digital​

Access digital pins

FunctionDescription
DRead(pin, pull)pin: pin number - pull: 0 = none, 1 = up, 2 = down - Returns: 1 = high or 0 = low.
DWrite(pin, state)pin: pin number - state: 1 = high or 0 = low.

Analog​

Access pins in their analog format

FunctionDescription
VRead(pin)pin: pin number - Returns: voltage level of a pin. Only works on Analog pins.
PWrite(pin, power)pin: pin number - power: power level, 0 to 1. 0.5 is 50% for example.

Beep​

Generate tones. This is a blocking function that works on all pins.

FunctionDescription
Beep(pin, frequency, duration)pin: Pin number - frequency: Hz max 10KHz - duration: In milliseconds.

Beep​

Generate tones. This is a non-blocking function that works on PWM pins.

FunctionDescription
Melody().name="melody", 4, api_melody

Button​

Use interrupts to keep track of pins. Only available on interrupt capable pins, see special pins above!

FunctionDescription
BtnEnable(pin, enable)pin: Pin number - enable: 1 = Enable 0 = Disabled
BtnUp(pin)pin: Pin number - Returns: 1 if button pin went high, 0 on second call till next transition.
BtnDown(pin)pin: Pin number - Returns: 1 if button was pressed, 0 on second call till next transition.

Frequency​

Sets a frequency (PWM) parameters to a pin. Only works on pins that support PWM.

FunctionDescription
Freq(pin, frequency, duration, dutycycle)Set PWM pin to frequency for duration in milliseconds at dutycycle where 1 is 100%, 0.2 is 20%...etc.

Touch​

Read capacitive touch by charging a pin and then reading the discharge time in microseconds.

FunctionDescription
Touch(pin, charge_t, charge_s, timeout)Charges pin with charge state charge_s for charge time charge_t in milliseconds then read how long it takes for pin to discharge, returning value in microseconds. timeout is used in case the pin didn't change its state.

Digital Control​

Distance​

Read distance from ultrasonic distance sensors.

FunctionDescription
Dist(trig, echo)Returns distance in centimeters from ultrasonic distance sensor connected to trig and echo pins.

Servo Motors​

Control servo motors.

FunctionDescription
ServoSet(pin, degree)pin: pin number - degree: 0 to 180

On Pins with PWM feature, the servo control is done in hardware, and it is very accurate. All other pins support Servo but give estimate on servo steps.

I2C​

Access the I2C bus.

FunctionDescription
I2CCfg(baudrate)baudrate: Set the I2C clock speed in khz, typically 100K or 400K, and 1M in some cases.
I2cBytes(address, arrayWrite, writeCount, arrayRead, readCount)address: I2C slave address - arrayWrite: array to send, writeCount: number of bytes to write, arrayRead: array to read,readCount: Number bytes to read

SPI​

Access the SPI bus.

FunctionDescription
SpiCfg(mode, frequency)mode: 0 to 3 - frequency: Set the desired frequency in kilohertz, ranging 200KHz to 20MHz
SpiByte(byte)byte: Data to write 0 to 255 - Returns: Read byte value
SpiWBytes(array, offset, count)array: Write an array to SPI bus, at offset with count

Serial​

Read and write serial data

FunctionDescription
SerCfg(baud, rxsize)????????????????????????
SerRead()Returns: a byte from the receive fifo. This function will not block and will return zero if it finds no data.
SerReads(array, offset, count, timeout)Block the system till it read a count of bytes to place in the byte array at offset. It will terminate and return if no data received in timeout milliseconds. Returns: How many bytes it read successfully.
SerWrite(data)???????????????????
SerWrites(array, offset, count)????????????????????????
SerB2R()Returns: How many bytes are available in the receive buffer.
SerDisc()Discard the receive buffer.

DMX​

Read and write from DMX devices. Must call SerCfg(250000, 513) before using these functions.

FunctionDescription
DmxW(channels, offset, count)Write channel byte array at offset and count.
DmxU()Enable receive to update internal buffer with incoming channels.
DmxRdy()Returns: 1 when there is a frame that was requested by SerDmxU()
DmxR(channel)Returns: Value of specific channel.
tip

This feature only support DMX frames with Start Code zero.

Infrared​

Receive and decode signals from Infrared remotes.

FunctionDescription
IrEnable(pin,enable)pin: on supported pins enable: 1 = enable or 0 = disable
IrRead()Returns: key press value 0 to 24

Temperature​

Read temperature from a specific sensor

FunctionDescription
Temp(pin, type)pin: The pin where the sensor is connected. type: see table below
IDType
0Internal CPU temperature (pin is ignored)
1DHT11 / DHT12
2DHT21 / DHT22
3One wire DS18B20 sensors

Humidity​

Read humidity from a specific sensor

FunctionDescription
Humid(pin, type)pin: The pin where the sensor is connected. type: see table below
IDType
0reserved
1DHT11 / DHT12
2DHT21 / DHT22

Graphics​

DUELink firmware include a flexible graphics facilities, detailed on the Graphics Engine page.

Graphics Config​

Configure the Graphics Engine.

FunctionDescription
GfxCfg(...)See the Graphics Engine page.

Drawing​

FunctionDescription
Clear(color)color: color value
Circle(color, x, y, radius)color: Color value - x: x center - y: y center - radius: radius
Fill(color, x, y, width, height)color: Color value - x: Starting x point - y: Starting y point - width: Rectangle width - height: Rectangle height
Img(array, x, y, transform)array: Image array - x: x position - y: y position - transform: 0 = none, 1 = Flip horz, 2 = Flip vert, 3 = 90 deg, 4 = 180 deg, 5 = 270 deg
ImgS(array, x, y, transform, scaleWidth, scaleHeight)Same as Img() but adds scaleWidth and scaleHeight
Line(color, x1, y1, x2, y2)color: Color values - x1: Starting x point - y1: Starting y point - x2: Ending x point - y2: Ending y point
Pixel(color, x, y)color: Color value - x: x position - y: y position
Rect(color, x, y, width, height)color: Color value - x: Starting x point - y: Starting y point - width: Rectangle width - height: Rectangle height
Text("text", color, x, y)text: string message, use Str() to convert variable to string - color: Color value - x: x position - y: y position
TextT("text", color, x, y)Tiny 5px text: string message, use Str() to convert variable to string - color: Color value - x: x position - y: y position
TextS("text", color, x, y, scaleWidth, scaleHeight)same as Text() but adds scaleWidth and scaleHeight
Show()Shows graphics buffer on the configured device.

File System​

DUELink firmware include a complete file system graphics facilities, detailed on the File System page.

Mount​

Mount and initialize the file system.

FunctionDescription
FsMount(cs, baud, max_handle)Enable file system to use SD and Micro SD cards on SPI bus with cs chip-select pin at baud with maximum allowed file handles max_handle. Returns 0 if success.
FsUnMount(...)Release file system. Returns 0 if success.

Operations​

tip

Unless otherwise stated, all functions return 1 on success or negative error code.

FunctionDescription
FsOpen(path, mode)Returns a file handle on success. 0 or negative on failure.
FsClose(handle)Close file and release handle.
FsWrite(handle, data, offset, count)Write from data array starting at offset and for count bytes. Returns how many bytes were read, negative error code otherwise.
FsRead(handle, data, offset, count)Read into data array starting at offset and for count bytes. Returns how many bytes were read, negative error code otherwise.
FsSync(handle)Sync file with any buffered data.
FsSeek(handle, offset)change the location in the handle file on where to read or write.
FsTell(handle)Returns the current location being access in handle file.
FsMkdir(path)Create a new directory at path.
FsDelete(path)Delete file or directory at path.
FsFind(path)Find if existing name is available. Returns 1 if found directory, 2 if file, failed otherwise.
FsSize(path)Returns path file size, negative number on failure, 0 if file not found.

File Open Modes | ID | mode | --- | | 0x01 | Read only | 0x02 | Write only | 0x00 | Open existing file | 0x04 | Create new | 0x08 | Create always | 0x10 | Open always | 0x30 | Open append

tip

Combine multiple modes, like use FsOpen("/test.txt", 0x02|0x0x30) to pen a file for write and append if existing.

Scheduler​

See Scheduler for details.

FunctionDescription
SStart("func", timeout, count)func: Function to schedule - timeout: how often to run (in milliseconds) - count: how many times to repeat, with -1 running forever.
SStat("func")func: Name of function to check status on - Returns: count remain to run. -1 when forever, -2
SAbort("func")func: Abort scheduler for named function.

Interrupts​

See Interrupts for details.

FunctionDescription
IStart("func", pin, trigger, pull)func: Function to trigger on interrupt - pin: the pin to enable for interrupt - trigger: How to enable interrupt: 0 - falling, 1 - rising, 2 - rising and falling - pull Enable pull resistors: 0 - pull down, 1 - pull up, 2 - no pull
IStat("func")func: Status of named function - Returns: 0- no interrupt, 1- active interrupt.
IAbort("func")func: Abort interrupt for named function.

Device Addressing​

These functions are needed for helping with Dasylink devices chained through Downstream connections.

FunctionDescription
Sel(device)device: Select one of the devices in the chain, with first device being address 1. Broadcast to all device is possible when using 0.
GetAdd()Returns: address of current selected device. With broadcast, all devices respond!
SetAdd(add)add Set the device's address. This command has chain reaction where each device sets teh next one. This is a special command and should not be used.

Math and Converters​

Random Numbers​

Return a random number

FunctionDescription
Rnd(max)max: maximum value of random number - Returns: random number between 0 and max

Trig​

FunctionDescription
Cos(number)Returns: cosine of its argument
Sin(number)Returns the sine value of its argument
Tan(number)Returns the tangent of it's argument

Square Root​

FunctionDescription
Sqrt(number)Returns the square root value of it's argument

Truncate​

FunctionDescription
Trunc(number)Returns the truncated value of it's argument

Is Not a Number​

FunctionDescription
IsNaN(number)Returns 1 if a valid number, 0 invalid.

String Numbers​

Convert a number to a string. Useful for Graphics

FunctionDescription
Str(number)Returns number as string

Format​

Formats multiple arguments into a single string

FunctionDescription
Fmt(...)arguments work just like they do with Print()

Misc​

Print​

Print to output.

FunctionDescription
Print("text" or variable)returns the value of it's argument
PrintLn("text" or variable)returns the value of it's argument with line breaks

Print() accepts multiple argument, separated by commas, Print("Value is: ", _v)

tip

Print is invoked automatically when a function returns a value that is not consumed. Running Dread(1,1) in immediate mode will show 0 or 1.

AsIO​

Enable Asynchronous IO. Detailed on the Engine main poage.

FunctionDescription
AsIo(enable)enable: 1 = Enable, 0 = Disable

Reset​

Reset or clear the system.

FunctionDescription
Reset(mode)mode: 0 = system reset, 1 = Enter DUELink loader, 2= switch LDR pin to BOOT0 pin, 3= Switch LDR pin to BOOT0 and erase the entire chip (factory reset!).

Echo​

Echoes back what is received, useful when using a terminal program.

FunctionDescription
Echo(state)state: 1 = Enable or 0 = Disable

Ticks​

Read system time, useful to run things at specific time or create timeouts.

FunctionDescription
TickMs()Read system ticks in milliseconds
TickUs()Read system ticks in microseconds

Version​

Read hardware and software version.

FunctionDescription
Version()Returns firmware version, device ID, and loader version, in this format: GHI Electronics DUELink v01.03:BB01:01.00.

Wait​

Stop running for milliseconds

FunctionDescription
Wait(duration)duration: in milliseconds

Dump Array​

FunctionDescription
DumpArray(array)array: Dumps array content.

Read VCC​

FunctionDescription
ReadVCC()Returns: Current system voltage.

One Time Programmable​

Write and read from a special memory type that can be written only once and can never be erased!

FunctionDescription
OtpR(addr)addr Address to read a single byte from - Returns: value of selected address.
OtpW(addr, val)addr Address to write to, must be multiple of 8 - val: value to write to an address. Must be an array size of 8 or multiple of 8 (16, 24...) and can only be done once, for specific address - Returns: 1 on success, 0 if failed.
tip

Use byte arrays. If float array is used then only the first byte of each element is written.
