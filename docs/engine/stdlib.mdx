
# Standard Library

---

These standard library functions are built in natively into the firmware and are available on all systems. You will likely not use these directly, but use the simpler wrapper methods provided by the individual modules.

:::tip
These are particularly useful for using [Mainboards](../catalog/mainboards) or or for making custom modules.
:::

## Pin Control

### LED

Control the on-board STAT LED, found on every single DUELink module.

| Function | Description
| --- | ---
| `LED(high, low, count)` | `high`: duration on in milliseconds, `low`: duration off in milliseconds, `count`: number of times to repeat, -1 to run forever.

### Digital

Access digital pins

| Function | Description
| --- | ---
| `DRead(pin, pull)` | `pin`: pin number - `pull`: 0 = none, 1 = up, 2 = down - Returns: 1 = high or 0 = low.
| `DWrite(pin, state`) | `pin`: pin number - `state`: 1 = high or 0 = low.

### Analog

Access pins in their analog format

| Function | Description
| --- | ---
| `VRead(pin)` | `pin`: pin number - **Returns**: voltage level of a pin. Only works on Analog pins.
| `PWrite(pin, power)` | `pin`: pin number - `power`: power level, 0 to 1. 0.5 is 50% for example.

### Beep

Generate tones. This is a blocking function that works on all pins.

| Function | Description
| --- | ---
| `Beep(pin, frequency, duration)` | `pin`: Pin number - `frequency`: Hz max 10KHz - `duration`: In milliseconds.

### Beep

Generate tones. This is a non-blocking function that works on PWM pins.

| Function | Description
| --- | ---
| `Melody()` | `.name="melody", 4, api_melody`

### Button

Use interrupts to keep track of pins. Only available on interrupt capable pins, see special pins above!

| Function | Description
| --- | ---
| `BtnEnable(pin, enable)` |  `pin`: Pin number - `enable`: 1 = Enable 0 = Disabled
| `BtnUp(pin)` |  `pin`: Pin number - **Returns**: 1 if button pin went high, 0 on second call till next transition.
| `BtnDown(pin)` |  `pin`: Pin number - **Returns**: 1 if button was pressed, 0 on second call till next transition.

| Function | Description

## Graphics

DUELink firmware include a flexible graphics facilities, detailed on the [Graphics Engine](./graphics) page.

### Graphics Config

Configure the Graphics Engine.

| Function | Description
| --- | ---
| `GfxConfig(...)` | See the [Graphics Engine](./graphics) page.

### Drawing

| Function | Description
| --- | ---
| `Clear(color)` | `color`: color value
| `Circle(color, x, y, radius)` | `color`: Color value - `x`: x center - `y`: y center - `radius`: radius
| `Fill(color, x, y, width, height)` |  `color`: Color value - `x`: Starting x point - `y`: Starting y point - `width`: Rectangle width - `height`: Rectangle height
| `Img(array, x, y, transform)` | `array`: Image array - `x`: x position - `y`: y position - `transform`: 0 = none, 1 = Flip horz, 2 = Flip vert, 3 = 90 deg, 4 = 180 deg, 5 = 270 deg
| `ImgS(array, x, y, transform, scaleWidth, scaleHeight)` | Same as `Img()` but adds `scaleWidth` and `scaleHeight`
| `Line(color, x1, y1, x2, y2)` | `color`: Color values - `x1`: Starting x point - `y1`: Starting y point - `x2`: Ending x point - `y2`: Ending y point
| `Pixel(color, x, y)` | `color`: Color value - `x`: x position - `y`: y position
| `Rect(color, x, y, width, height)` | `color`: Color value - `x`: Starting x point - `y`: Starting y point - `width`: Rectangle width - `height`: Rectangle height
| `Text("text", color, x, y)` | `text`: string message, use Str() to convert variable to string - `color`: Color value - `x`: x position - `y`: y position
| `TextT("text", color, x, y)` | Tiny 5px `text`: string message, use Str() to convert variable to string - `color`: Color value - `x`: x position - `y`: y position
| `TextS("text", color, x, y, scaleWidth, scaleHeight)` | same as `Text()` but adds `scaleWidth` and `scaleHeight`
| `Show()` | Shows graphics buffer on the configured device.

## Scheduler

See [Scheduler](./scheduler) for details.

| Function | Description
| --- | ---
| `SStart("func", timeout, count)` | `func`: Function to schedule - `timeout`: how often to run (in milliseconds) - `count`: how many times to repeat, with -1 running forever.
| `SStat("func")` | `func`: Name of function to check status on - **Returns**: count remain to run. -1 when forever, -2
| `SAbort("func")` | `func`: Abort scheduler for named function.

---

## Interrupts

See [Interrupts](./interrupt) for details.

| Function | Description
| --- | ---
| `IStart("func", pin, trigger, pull)` | `func`: Function to trigger on interrupt - `pin`: the pin to enable for interrupt - `trigger`: How to enable interrupt: 0 - falling, 1 - rising, 2 - rising and falling - `pull` Enable pull resistors: 0 - pull down, 1 - pull up, 2 - no pull
| `IStat("func")` | `func`: Status of named function - **Returns**: 0- no interrupt, 1- active interrupt.
| `IAbort("func")` | `func`: Abort interrupt for named function.

## Device Addressing

These functions are needed for helping with [Dasylink](./daisylink) devices chained through [Downstream](../interface/downstream) connections.

| Function | Description
| --- | ---
| `Sel(device)` | `device`: Select one of the devices in the chain, with first device being address 1. Broadcast to all device is possible when using 0.
| `GetAdd()` | **Returns**: address of current selected device. With broadcast, all devices respond!
| `SetAdd(add)` | `add` Set the device's address. This command has chain reaction where each device sets teh next one. **This is a special command and should not be used.**

## Misc

### Print

Print to output. 

| Function | Description
| --- | ---
| `Print("text" or variable)` | returns the value of it's argument
| `PrintLn("text" or variable)` | returns the value of it's argument with line breaks

`Print()` accepts multiple argument, separated by commas, `Print("Value is: ", _v)`

:::tip
Print is invoked automatically when a function returns a value that is not consumed. Running `Dread(1,1)` in immediate mode will show 0 or 1.
:::

### AsIO

Enable Asynchronous IO. Detailed on the [Engine](./intro.mdx) main poage.

| Function | Description
| --- | ---
| `AsIo(enable)` | `enable`: 1 = Enable, 0 = Disable

### Reset

Reset or clear the system.

| Function | Description
| --- | ---
| `Reset(mode)` | `mode`: 0 = system reset, 1 = Enter DUELink loader, 2= switch LDR pin to BOOT0 pin, 3= Switch LDR pin to BOOT0 and erase the entire chip (factory reset!).

### Echo

Echoes back what is received, useful when using a terminal program.

| Function | Description
| --- | ---
| `Echo(state)` | `state`: 1 = Enable or 0 = Disable

### Ticks

Read system time, useful to run things at specific time or create timeouts.

| Function | Description
| --- | ---
| `TickMs()` | Read system ticks in milliseconds
| `TickUs()` | Read system ticks in microseconds

### Version

Read hardware and software version.

| Function | Description
| --- | ---
| `Version()` | **Returns** firmware version, device ID, and loader version, in this format: `GHI Electronics DUELink v01.03:BB01:01.00`.


### Wait

| Function | Description

| `Wait(duration)` | `duration`: in milliseconds

### Dump Array

| Function | Description

### CPU Temperature

| Function | Description
| --- | ---
| `Wait()` | **Returns**: Current internal CPU temperature. 

### Read VCC

| Function | Description
| --- | ---
| `ReadVCC()` | **Returns**: Current system voltage.

### One Time Programmable

Write and read from a special memory type that can be written only once and can **never** be erased!


| Function | Description
| --- | ---
| `OtpR(addr)` | `addr` Address to read from - **Returns**: value of selected address
| `OtpW(addr, val)` | `addr` Address to write to - `val`: value to write to an address. Can only be done once for specific address.

Note that the address must be multiple of 4.
