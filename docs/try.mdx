# Type Your First Command

Plug a DUELink module into USB, then type a command below to see it work.

> **Live terminal coming soon.** For now, try it on [demo.duelink.com](https://demo.duelink.com/).

## Command to try

Every module supports one universal command:

- `StatLed(on, off, count)` — blink the on-board status LED

All three numbers are in **milliseconds**. `StatLed(200, 200, 5)` turns the LED on for 200ms, off for 200ms, and repeats 5 times.

That works on **any** DUELink module — use it to confirm your connection before anything else.

Each module has **additional** commands of its own (relays, sensors, displays, and so on). Those are listed on that module's **Commands** tab in the [catalog](./catalog/products).

---

## How modules work via text commands

Every DUELink module runs its own tiny firmware that listens for **plain ASCII text** over USB, I2C, or UART. When the module recognizes a command, it runs it. When the command returns a value — like a sensor reading — the module sends the answer back as text, terminated with a carriage return.

So confirming a connection literally looks like this on the line:

```
StatLed(200,200,5)
OK
```

Some commands return a value — for example, a distance module might answer `12.4` after you send its read command. Those commands are **different on every module**; see that module's **Commands** tab in the catalog.

That's the entire protocol. No register maps, no SPI dance, no datasheet drill-down. The command names are the same whatever host language you write in and whatever interface you connect over — USB, I2C, or UART.

`StatLed(on, off, count)` is the one command on **every** module. Everything else is module-specific — a relay module has `RelayOn(n)`, a display has `DrawLine(...)` and `Show()`, and so on.

## How this page sends commands

The terminal above is just a small piece of JavaScript that runs entirely in your browser. It uses the standard **Web Serial API** (`navigator.serial`) to open the USB port your DUELink module is connected to, then it does exactly what a desktop terminal would do:

1. Reads what you type into the input box.
2. Writes those bytes to the serial port, followed by a carriage return.
3. Reads bytes back from the port until a carriage return arrives.
4. Prints the response.

There's no DUELink-specific protocol magic. No server. Nothing leaves your machine. It's a few dozen lines of plain JavaScript talking to the same text command interface that any host language can use.

That's why **the libraries on the [Use from Your Code](./start) page do the same thing in Python, JavaScript, .NET, Arduino, and MicroPython** — they wrap a serial / I2C / UART transport and call `Engine.ExecuteCommand("...")`. If you can type `StatLed(200,200,5)` here and watch the LED blink, you can do the exact same thing from your own code in five lines.
