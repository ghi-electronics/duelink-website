# WiFi

Having an internet connection is essential in many projects. Some of the [supported hardware](../hw/intro) options already have WiFi. This can for example be a [PC](../hw/pc-laptop) or a [Raspberry Pi](../hw/raspberry-pi) connected to a [Cloud](./cloud) like Azure.

<div style={{ textAlign: 'center'}}>
![Azure Cloud](pathname:///img/azure-cloud.webp)
</div>

DUELink also includes the [Wireless ESP32](../products/wlesp32-a) module, which serves two different use cases.

<div style={{ textAlign: 'center'}}>
![Wireless ESP32](pathname:///img/catalog/wlesp32-a-1.webp)
</div>

You can use this module as a `Wireless Interface` to a daisylink of modules. This can for example be a PC that is controlling a [Daisylink](../engine/daisylink) of modules using [Python](../language/python) over a wireless Bluetooth connection.

<div style={{ textAlign: 'center'}}>
![Wireless daisylink](pathname:///img/wireless-daisylink.webp)
</div>

Or use it to add a wireless connection to any DUELink setup, via AT commands or a data gateway.

<div style={{ textAlign: 'center'}}>
![Wireless ESP32 no host](pathname:///img/wireless-daisylink-addr-nohost.webp)
</div>

Both cases ar explained on the [Wireless Interface](../interface/wireless) page.
